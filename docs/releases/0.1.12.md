# manyfold 0.1.12

Release date: 2026-05-03

## Summary

manyfold 0.1.12 is a backward-compatible patch release for disk-backed route
memory validation. It extends `Memory(path)` startup checks so malformed JSONL
records with wrong field types or invalid base64 payloads fail immediately with
clear path-and-line diagnostics.

## What Changed

### Hardened memory record field validation

`Memory(path)` now validates required JSONL record fields after confirming they
exist. Route, schema id, and payload fields must be strings; sequence and schema
version fields must be integers; and control epochs must be integers or null.

Before this release, a hand-edited or partially corrupted memory file could
carry type-invalid records into later resume or duplicate-filter paths, producing
less actionable failures.

### Validated encoded payload bytes at load time

Memory records now validate `payload_b64` with strict base64 decoding while the
file is loaded. Invalid payload text raises a `ValueError` naming the memory file
and line, instead of failing later during record decoding.

### Added regression coverage

The component tests now cover invalid memory field types and invalid base64
payloads, asserting that both errors include the affected path and line.

## Why This Matters

`Memory` is a durable recovery helper. Bad records should be reported at the
point where the file is opened, with enough context for operators or tests to
repair the exact JSONL line. This keeps recovery failures deterministic and
prevents malformed disk state from leaking into graph replay.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- The memory JSONL format is unchanged for valid files.
- Existing valid memory files continue to load normally.
- Invalid memory files now fail earlier with documented `ValueError` messages.

## Versioning Decision

This release is tagged as `0.1.12` rather than `0.2.0` because it tightens
validation for an existing durable recovery path without changing public APIs or
valid file formats.
