# manyfold 0.1.8

Release date: 2026-05-03

## Summary

manyfold 0.1.8 is a backward-compatible patch release for local sensor buffering.
It fixes `BoundedRingBuffer(overflow="latest")` loss accounting so the public
`dropped` counter reports every item cleared when a full buffer is replaced by
the newest sample.

## What Changed

### Fixed latest-overflow drop accounting

`BoundedRingBuffer` supports a `latest` overflow mode for sources where only the
newest sample should survive bursts. When a full latest-mode buffer receives a
new item, it clears all staged items and stores the replacement.

Before this release, that replacement path incremented `dropped` by `1` even
when it discarded a full buffer of multiple samples. A capacity-3 latest buffer
that replaced `(1, 2, 3)` with `4` now reports `dropped == 3`, matching the
number of samples that were actually discarded.

### Added regression coverage

The sensor IO tests now cover multi-item replacement in latest overflow mode and
assert both the retained value and the full dropped count.

## Why This Matters

Sensor adapters and local acquisition loops use `dropped` as loss telemetry. An
under-reported count can hide backpressure problems during bursts, especially
when latest-mode buffers are intentionally used to keep consumers current while
discarding stale samples.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- FIFO, reject, drop-oldest, and drop-newest behavior is unchanged.
- The only behavior change is more accurate accounting for latest-mode buffer
  replacement.

## Versioning Decision

This release is tagged as `0.1.8` rather than `0.2.0` because it fixes telemetry
for an existing public API without changing the intended API shape.

## Suggested GitHub Release Notes

### Highlights

- Fixed `BoundedRingBuffer(overflow="latest")` to count every discarded item
  when replacing a full buffer.
- Kept existing latest-mode retention behavior: the newest item is retained and
  stale buffered items are cleared.
- Added regression coverage for multi-item latest replacement.
- Bumped package version to `0.1.8`.

### Detailed Notes

manyfold `0.1.8` fixes latest-mode sensor buffer telemetry. Full-buffer
replacement now increments `dropped` by the number of cleared samples instead of
always reporting a single dropped item.
