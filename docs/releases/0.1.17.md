# manyfold 0.1.17

Release date: 2026-05-03

## Summary

manyfold 0.1.17 is a backward-compatible patch release that groups the
post-0.1.16 mainline work into a new PyPI build. It expands the graph-facing
Python API with route pipelines and diagram rendering, tightens a few runtime
edge cases, and refreshes the documentation around the logical board model.

## What Changed

### Added graph route pipelines

`Graph.observe(route)` now returns a `RoutePipeline` that supports fluent
graph-visible `map`, `filter`, `callback`, and route connection nodes. Pipelines
can be extended with `Graph.register_pipeline_operation(...)`, and installed
nodes participate in diagram metadata so the processing topology is visible
alongside routes.

### Added topology diagram rendering

The graph can render Mermaid flowcharts from route topology and registered
diagram nodes. Diagram metadata is descriptor-driven, so callers can group
routes by stable fields such as owner or layer without manually assembling a
visual model.

### Improved reactive helper performance and compatibility

The reactive-thread helper surface gained follow-up cleanup after 0.1.16:
latency snapshots avoid repeated sorting, and internal Rx imports are routed
through Manyfold's private compatibility facade rather than a public top-level
`manyfold.rx` module.

### Hardened runtime correctness

- Route schema coercion now applies both `schema_id` and `version` overrides
  when callers pass an existing `Schema`.
- `Graph.observe(...).subscribe(...)` now rolls back subscriber counts and
  active subscriber audit entries if latest-value replay raises in the caller's
  observer.
- Guarded acknowledgement checks were refactored around shared helpers.
- Delimited message buffer draining avoids an unnecessary exception path.

### Improved embedded and catalog surfaces

Embedded runtime profiles now validate local processing capabilities more
explicitly, and the lego catalog caches query indexes for faster repeated role
and layer lookups.

### Refreshed documentation

The README and topology image were updated to present Manyfold as a logical
board for routed application state, payloads, write shadows, and debug views.
The documentation set now also includes focused onboarding, usage, and
performance guides for getting started without keeping all of that detail in
the README.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing route, schema, component, and reactive-thread APIs remain compatible.
- The new route pipeline and diagram helpers are additive.
- The version bump only packages changes that were already merged to `main`
  after `v0.1.16`.

## Versioning Decision

This release is tagged as `0.1.17` because it packages several additive
mainline improvements and small correctness fixes without breaking existing
APIs.

## Suggested GitHub Release Notes

### Highlights

- Added fluent graph route pipelines with graph-visible map, filter, callback,
  and route sink nodes.
- Added Mermaid topology diagram rendering for routes and registered processing
  nodes.
- Fixed replay-subscription cleanup when an observer raises during latest-value
  replay.
- Improved reactive-thread helper internals, embedded validation, guarded ack
  checks, delimited-buffer draining, and lego catalog query performance.
- Refreshed the README, logical board image, and onboarding/usage/performance
  docs.

### Detailed Notes

manyfold `0.1.17` packages the post-`0.1.16` work already merged to `main` into
a new PyPI release. The largest user-facing addition is the graph route pipeline
surface, which lets applications wire typed route observations through visible
processing nodes and then render the resulting topology.
