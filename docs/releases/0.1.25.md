# manyfold 0.1.25

Release date: 2026-05-06

## Summary

manyfold 0.1.25 is a backward-compatible patch release for fixes that landed
after the `0.1.24` tag. It tightens mailbox validation, exposes the complete
native producer-kind surface, preserves empty inline payloads, stabilizes repo
path handling in example tests, prevents mixed text/byte sensor message
corruption, rejects ambiguous consensus JSON booleans, stabilizes graph replay
ordering, and clarifies average-window behavior.

## What Changed

### Hardened runtime behavior

- Mailbox descriptors now reject zero capacity during validation instead of
  accepting a configuration that cannot make progress.
- Opening an inline payload now preserves empty byte payloads instead of
  treating them as missing external payloads.
- Native graph producer-kind exports now cover every producer kind exposed by
  the Rust runtime.
- Text-delimited message buffers now reject appending decoded text while a
  partial UTF-8 byte sequence is pending, preventing silent message corruption.
- Consensus JSON decoding now rejects string boolean values instead of treating
  non-empty strings as truthy flags.
- Readable graph ports are subscribed before replay so observers do not miss
  replayed values during setup.
- Removed route pipeline topology is cleaned up so stale route edges do not
  survive after removal.

### Stabilized validation and examples

- Repository import-path ordering is stable when tests load examples through
  both the installed package and source checkout helpers.
- Mailbox bridge example topology expectations now match sorted topology query
  output.
- Topology and coalescing tests are less sensitive to ordering and scheduler
  jitter while still covering the same behavior.

### Documentation

- Average window documentation now calls out that oversized windows average the
  available values.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing route, graph, component, consensus, sensor, and schema APIs remain
  compatible.
- Validation is stricter for invalid zero-capacity mailbox descriptors.

## Versioning Decision

This release is tagged as `0.1.25` because it packages several patch-level
runtime and validation fixes that landed immediately after `0.1.24`, including
empty inline payload preservation and zero-capacity mailbox rejection.
