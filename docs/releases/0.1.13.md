# manyfold 0.1.13

Release date: 2026-05-03

## Summary

manyfold 0.1.13 is a backward-compatible patch release for sensor JSON schema
decoding. It tightens byte payload validation so malformed base64 in sensor
sample values, sensor event raw bytes, and nested event data bytes fails with a
clear `ValueError`.

## What Changed

### Hardened sensor schema byte decoding

`sensor_sample_schema(...).decode(...)` now strictly validates the JSON `value`
field before passing bytes to the wrapped value schema. Invalid encoded sample
payloads now fail as `ValueError: value must be valid base64`.

`sensor_event_schema().decode(...)` now uses the same strict path for `raw`
bytes and nested `{"__bytes_b64__": ...}` values restored from event data or
metadata.

Before this release, malformed base64 could either be accepted by Python's
permissive decoder or escape as a low-level decoding exception, which made bad
sensor transport records harder to diagnose.

### Added regression coverage

The sensor IO tests now cover invalid base64 in sample values, event raw bytes,
and nested event byte fields.

## Why This Matters

Sensor JSON schemas are a persistence and transport boundary. Corrupt byte
payloads should be rejected deterministically before they reach user value
decoders or replay pipelines.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- Valid existing sensor JSON payloads decode unchanged.
- Malformed base64 payloads now raise clearer `ValueError` diagnostics.

## Versioning Decision

This release is tagged as `0.1.13` rather than `0.2.0` because it tightens
validation for existing schema decoders without changing public APIs or valid
payload formats.
