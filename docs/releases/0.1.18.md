# manyfold 0.1.18

Release date: 2026-05-03

## Summary

manyfold 0.1.18 is a backward-compatible patch release that makes execution
placement a graph-visible node concept. Route pipelines can now declare whether
following work should run on the main frame thread, the shared background pool,
a named worker pool, or an isolated dedicated thread.

## What Changed

### Added graph-visible thread placement

`RoutePipeline` now supports fluent placement methods:

- `.on_main_thread()`
- `.on_background_thread()`
- `.on_pooled_thread()`
- `.on_isolated_thread()`

The selected placement applies to downstream `map`, `filter`, `callback`, route
connections, and direct subscription delivery until another placement is chosen.
Individual `CallbackNode`, `MapNode`, and `FilterNode` values also expose the
same placement helpers for node-local overrides.

### Integrated placement with existing schedulers

Main-thread placement uses the existing frame-thread queue in
`manyfold.reactive_threads`. Background and pooled placement use the shared
worker schedulers, while isolated placement creates a dedicated event-loop
scheduler owned by the installed connection and disposes it when the connection
is removed.

### Exposed placement in diagram metadata

Registered diagram nodes now carry their selected `NodeThreadPlacement`, and
diagrams can group nodes by `thread` so scheduling boundaries show up alongside
route topology.

### Refreshed usage documentation

The usage guide now includes a placement example, and the README lists thread
placement as a modeled graph concern.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing route, schema, component, and reactive-thread APIs remain compatible.
- The new `NodeThreadPlacement` type and fluent placement helpers are additive.
- Existing pipelines keep their current inline delivery behavior unless callers
  opt into explicit placement.

## Suggested GitHub Release Notes

### Highlights

- Added `NodeThreadPlacement` for graph-visible main, background, pooled, and
  isolated node execution.
- Added `.on_main_thread()`, `.on_background_thread()`, `.on_pooled_thread()`,
  and `.on_isolated_thread()` to route pipelines and graph-visible nodes.
- Applied placement to downstream map/filter/callback/route/subscription
  delivery.
- Added diagram metadata and grouping support for node thread placement.

### Detailed Notes

manyfold `0.1.18` makes thread placement part of the graph model instead of an
implicit callback concern. Applications can now describe scheduling boundaries
in the same fluent pipeline surface they use for processing nodes, and those
boundaries remain inspectable through registered node metadata and diagrams.
