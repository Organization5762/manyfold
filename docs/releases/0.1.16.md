# manyfold 0.1.16

Release date: 2026-05-03

## Summary

manyfold 0.1.16 is a backward-compatible patch release that adds a
`manyfold.reactive_threads` migration surface for applications currently using
`heart.utilities.reactive_threads`.

## What Changed

### Added reactive thread helpers

`manyfold.reactive_threads` now provides shared Rx scheduler helpers,
frame-thread delivery, latency snapshots, pipeline helpers, and test reset
support. The module uses Manyfold's private Rx facade and keeps the helper
surface out of the top-level `manyfold` namespace.

The shared scheduler worker counts can be configured with:

- `MANYFOLD_RX_BACKGROUND_MAX_WORKERS`
- `MANYFOLD_RX_BLOCKING_IO_MAX_WORKERS`
- `MANYFOLD_RX_INPUT_MAX_WORKERS`

For migration compatibility, the corresponding `HEART_RX_*` environment names
are still honored when the new `MANYFOLD_RX_*` names are not set.

### Added regression coverage

The new tests cover scheduler configuration, legacy environment fallback,
frame-thread queue draining, disposed queued deliveries, pipeline helpers, and
iterable sharing.

## Why This Matters

Applications can now replace `heart.utilities.reactive_threads` imports with
`manyfold.reactive_threads` while keeping the same core scheduling and
frame-thread handoff model.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No existing top-level `manyfold` exports changed.
- The new helper module is opt-in via `import manyfold.reactive_threads`.
- Existing `HEART_RX_*` worker-count settings continue to work during migration.
