# manyfold 0.1.15

Release date: 2026-05-03

## Summary

manyfold 0.1.15 is a backward-compatible patch release for disk-backed route
memory. It prevents records persisted for one schema id from being decoded
through another schema id that happens to share the same route identity and
version.

## What Changed

### Guarded Memory schema compatibility

`Memory.records(...)` and `Memory.resume(...)` now verify each persisted record's
`schema_id` and `schema_version` against the requested typed route before
decoding the payload. A mismatch raises a clear `ValueError` that names the
memory file, stored schema, route, and expected schema.

This closes a durable replay edge case where route display strings distinguish
schema version but not schema id, allowing two schemas with the same route
identity and version to collide at resume time.

### Added regression coverage

The component tests now cover recording bytes with one schema id and attempting
to read the same route identity through a different schema id. The test asserts
that memory rejects the mismatch before invoking the consumer decoder.

## Why This Matters

Route memory is a persistence boundary. Replaying bytes through the wrong typed
schema can silently corrupt application state when decoders accept broad payload
shapes, or fail later with an error that hides the real compatibility problem.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- Correctly matched memory records continue to decode as before.
- Mismatched route/schema reads now fail early instead of decoding with the
  requested route's schema.

## Versioning Decision

This release is tagged as `0.1.15` rather than `0.2.0` because it fixes a
backward-compatible persistence correctness issue without changing public APIs.

## Suggested GitHub Release Notes

### Highlights

- Validated Memory record schema id/version before payload decode.
- Added a regression test for same-route, different-schema persisted records.
- Bumped package version to `0.1.15`.

### Detailed Notes

manyfold `0.1.15` hardens disk-backed route memory so incompatible persisted
records fail with a targeted schema mismatch diagnostic before replay can decode
bytes through the wrong typed route.
