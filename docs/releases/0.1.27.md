# manyfold 0.1.27

Release date: 2026-05-09

## Summary

manyfold 0.1.27 is a backward-compatible patch release for fixes that landed
after 0.1.26. It tightens schema and runtime validation, stabilizes public and
private export surfaces, improves deterministic diagnostics, and cleans up
several runtime edge cases discovered by the test suite.

## What Changed

### Hardened schema and constructor validation

- Float schemas now reject booleans, non-numeric objects, malformed bytes, and
  non-finite payloads with a consistent finite-number error.
- Sensor JSON schemas now require JSON integers, strings, objects, and finite
  numbers in the fields that advertise those shapes.
- Health status JSON now rejects string stale flags and other malformed field
  types instead of coercing them.
- Consensus JSON and legacy tuple schemas now reject malformed integers,
  strings, booleans, arrays, replicated-log entries, construction inputs, and
  encoder payloads before publishing or storage.
- Route and schema construction now rejects empty identifiers and invalid native
  enum values earlier.
- Embedded profile, mailbox capacity, retry policy, batching, timing, graph
  policy, watchdog, interval join, time-windowing, logging, coalescing,
  registry, diagram name, isolated thread placement, and rate-matched sensor
  numeric inputs now fail fast when booleans, strings, floats, or other invalid
  types are supplied.

### Stabilized graph and runtime behavior

- Core graph connections are now idempotent.
- Closed envelopes are immutable.
- Route audit events preserve chronological order.
- Logging subscription rollback, failed module-load cleanup, coalescing state,
  empty `FileStore` directories, and non-finite latency samples are handled more
  predictably.
- Threshold filters now treat boolean transitions as value changes instead of
  comparing them numerically against a float threshold.
- Process-local `Schema.any` tokens are scoped by schema id and version, so
  unrelated schemas cannot decode each other's runtime-only tokens.
- Frame assembler slot ordering, catalog duplicate reports, lego dependency
  reports, and Mermaid diagram directions are deterministic.

### Improved exports, metadata, and tooling

- Top-level, component, helper, Rx operator, Rx testing, and private Rx facade
  exports are pinned or normalized for stable imports.
- Subprocess test `PYTHONPATH` handling and repository helper `sys.path`
  canonicalization are more stable.
- README example group headings, reference example metadata, project metadata,
  Cargo dependency ordering, and example export ordering are validated.
- The lego catalog now exposes dependency closure queries.
- RFC checklist path protocol handling is documented.
- Unsupported route schema errors are clearer.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing route, graph, schema, sensor, component, embedded, catalog, and Rx
  facade APIs remain compatible.
- The main behavior changes are stricter validation and clearer errors for
  malformed inputs that were previously coerced or failed later.

## Versioning Decision

This release is tagged as 0.1.27 because the accumulated fixes after 0.1.26 are
user-visible validation and runtime hardening improvements. They are compatible
patch changes, but there are enough of them to make a new PyPI version useful
rather than waiting for another grouped release.
