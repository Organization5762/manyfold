# manyfold 0.1.6

Release date: 2026-05-03

## Summary

manyfold 0.1.6 is a backward-compatible patch release for sensor text framing.
It prevents `DelimitedMessageBuffer` from accepting invalid UTF-8 delimiters in
text mode, avoiding an empty-delimiter drain loop after invalid delimiter bytes
were previously decoded with ignored errors.

## What Changed

### Fixed text delimiter validation

`DelimitedMessageBuffer(mode="text")` now validates that the configured
delimiter is valid UTF-8 during construction. Text-mode draining also decodes
the delimiter strictly instead of ignoring invalid bytes.

Before this release, a delimiter such as `b"\xff"` passed construction because
it was non-empty. During text-mode drain, the delimiter decoded to an empty
string with ignored errors. Searching for that empty delimiter could repeatedly
match without consuming buffered text.

After this release, invalid text delimiters raise:

```text
ValueError: text delimiter must be valid UTF-8
```

Byte-mode buffering is unchanged and continues to accept arbitrary non-empty
byte delimiters.

### Added regression coverage

The sensor I/O tests now cover invalid UTF-8 text delimiters so the buffer
cannot regress to silently ignoring delimiter decode failures.

## Why This Matters

Text-mode message framing is used for streaming sensor and peripheral adapters.
Invalid delimiter bytes should fail fast at configuration time instead of
turning into an empty delimiter at runtime, where appending data can hang the
caller.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- Valid UTF-8 text delimiters keep the same behavior.
- Byte-mode delimiter handling is unchanged.
- Invalid UTF-8 text delimiters now raise `ValueError` instead of being
  accepted.

## Versioning Decision

This release is tagged as `0.1.6` rather than `0.2.0` because it fixes a
runtime hang in an existing public API without changing the intended API shape.

## Suggested GitHub Release Notes

### Highlights

- Fixed `DelimitedMessageBuffer(mode="text")` invalid delimiter handling.
- Invalid UTF-8 text delimiters now fail fast with `ValueError`.
- Preserved byte-mode arbitrary byte delimiter support.
- Added regression coverage for the text delimiter edge case.
- Bumped package version to `0.1.6`.

### Detailed Notes

manyfold `0.1.6` fixes a sensor I/O framing bug where invalid UTF-8 delimiter
bytes in `DelimitedMessageBuffer(mode="text")` could decode to an empty string
when the drain loop ignored decode errors. Text-mode delimiters are now
validated once during construction and decoded strictly during draining, so
misconfigured text framing fails immediately instead of hanging while appending
data.
