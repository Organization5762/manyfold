# manyfold 0.1.31

Release date: 2026-05-10

## Summary

manyfold 0.1.31 is a backward-compatible patch release that packages the work
landed since 0.1.30 and fixes a top-level import regression for installations
that do not ship the repository `examples/` package. Importing `manyfold` now
keeps the reference example suite empty instead of failing during module import
when example metadata is unavailable.

This release also groups the current main-branch hardening work across graph,
sensor, component, primitive, embedded, and catalog APIs, along with the route
fanout payload-copy optimization, Rx facade/threading cleanup, and a handful of
example/catalog diagnostics improvements.

## What Changed

### Fixed package import fallback

- `manyfold.reference_examples` now treats missing example metadata as an empty
  reference suite instead of calling `reference_example_metadata(...)` during
  import.
- `import manyfold` remains available in stripped runtime environments that
  omit the repository `examples/` package.
- Added regression coverage that imports the module while forcing
  `ModuleNotFoundError("examples")`.

### Hardened public API construction and validation

- Graph validation now rejects invalid flow-policy overrides, invalid route-like
  objects, invalid pipeline node construction, invalid graph operator
  callbacks, invalid lazy payload source configuration, duplicate control loop
  installs, and invalid moving-average samples.
- Primitive and schema validation now rejects invalid direct `Keyspace`,
  `RouteIdentity`, `Source`, `Sink`, and schema codec-hook construction.
- Component and storage validation now rejects invalid `EventLog`,
  `SnapshotStore`, `Memory`, `FileStore`, and durable spool setup before
  runtime side effects occur.
- Sensor and embedded helpers now reject malformed sensor payloads, profiles,
  adapters, routes, callbacks, clocks, debug taps, backoff policies, and local
  graph node configuration earlier and with stable `ValueError` paths.
- Reactive-thread helpers now keep their public export surface ordered and
  reject invalid latency-recorder stream names and delay values earlier.
- Keyspace-backed stores now validate their store objects at construction time
  instead of failing later on filesystem-style operations.
- Lego catalog and example metadata validation now fail earlier with clearer
  diagnostics for malformed catalog content and lookups.

### Improved runtime behavior and packaging surface

- Route fanout avoids an eager payload copy on delivery paths that can reuse the
  existing bytes buffer.
- Graph and sensor helper wiring now route private Rx `Subject` usage through
  the local facade instead of reaching through the third-party package surface
  directly.
- Sensor IO and related helper exports are kept sorted so runtime, tests, and
  stubs observe the same stable public surface.
- Capped sensor backoff handling now clamps overflow instead of producing
  unstable retry behavior.
- Restored sensor JSON mappings are normalized deterministically, and the stats
  helper surface is exported explicitly.
- Reference export helpers, mailbox subprocess coverage, and TOML diagnostics
  were cleaned up to make example and tooling behavior more predictable.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing runtime setups that already include `examples/` keep the same
  reference example behavior.
- Runtime-only installs that omit `examples/` now import successfully instead
  of failing at module import time.
- Invalid public API construction paths now fail earlier with explicit
  validation errors instead of later native or attribute errors.

## Versioning Decision

This release is tagged as 0.1.31 because the import failure affects ordinary
package consumers that install the runtime without repository example sources,
and current `main` already contains a substantial set of compatible hardening
fixes since 0.1.30.
