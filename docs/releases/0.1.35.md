# manyfold 0.1.35

manyfold 0.1.35 is a backward-compatible patch release for long-running
runtime memory behavior.

## Fixes

- Native `GraphCore` now enforces route retention for default `latest_only` and
  `none` routes, bounding native history, payload storage, and route audit rows
  instead of retaining one entry per publish forever.
- Real native Python publishes now remember eager payload bytes before recording
  closed native envelopes, so typed `latest()` and `open_payload()` decode the
  latest payload instead of falling through to an empty inline payload.
- The existing priority-aware background stream handoff work is included in this
  package version.

- Native probe: 100 publishes through real `manyfold._manyfold_rust.Graph`
  retained native queue depth `1` and returned `b"sample-99"` from both
  `latest()` and `open_payload()`.

This release is tagged as `0.1.35` because the memory-retention fix is
compatible and release-blocking for long-running graph users.
