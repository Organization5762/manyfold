# manyfold 0.1.23

Release date: 2026-05-04

## Summary

manyfold 0.1.23 is a backward-compatible patch release that packages the
post-0.1.22 correctness and stability work. It focuses on graph replay ordering,
failed setup cleanup, durable component ordering, clearer diagnostics, and
example/package polish.

## What Changed

### Hardened graph replay and setup paths

- Fluent route pipelines now replay existing latest values through downstream
  map, filter, coalesce, log, callback, and route nodes.
- Graph operators subscribe before replaying latest values, so reentrant
  publishes during replay are not dropped.
- Pipeline, interval join, capacitor, resistor, watchdog, and durable spool
  setup paths now clean up earlier subscriptions or diagram nodes if a later
  subscription fails.
- `join_latest()` and `lookup_join()` treat decoded `None` as a present value
  instead of confusing it with an unseen internal sentinel.

### Improved durable component behavior

- Disk-backed memory appends are serialized so concurrent publishers do not race
  JSONL writes or duplicate suppression.
- Event log append persistence and committed-output publication remain under
  the same ordering lock, preserving durable index order for concurrent
  publishers.
- Snapshot stores now publish persisted `None` latest values as real route
  state.

### Stabilized APIs, diagnostics, and examples

- The native `CreditSnapshot` Python API exposes `largest_queue_depth`.
- Append-entry and consensus message schemas use JSON tuple payloads while still
  reading legacy delimited payloads.
- Path-based module loads restore the previous module if loading fails.
- Route missing-field errors and process-local schema token errors are clearer.
- Diagram debug route and node metadata output ordering is stable.
- Embedded validation catches duplicate catalog names, malformed bulk sensor
  route pairing, and reports issues in a stable order.
- The examples package facade and marble testing facade exports are stable.
- Catalog wrapper imports remain visible at module scope for import hygiene.
- `Graph` gained context-manager support.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing route, graph, component, sensor, and schema APIs remain compatible.
- New behavior is additive or fixes incorrect edge-case handling.
- Legacy delimited consensus payloads remain decodable.

## Versioning Decision

This release is tagged as `0.1.23` because it groups a substantial set of
post-0.1.22 patch fixes into one PyPI build without changing compatibility.

## Suggested GitHub Release Notes

### Highlights

- Fixed graph replay ordering so latest-value replay does not drop reentrant
  publishes.
- Cleaned up failed setup paths for graph operators, flow shapers, pipelines,
  and durable sensor spools.
- Serialized durable memory and event-log behavior under concurrent publishers.
- Published persisted `None` snapshots and preserved `None` join state.
- Stabilized diagnostics, diagrams, embedded validation, and example facades.

### Detailed Notes

manyfold `0.1.23` is a grouped patch release for the runtime hardening work
that landed after `0.1.22`. It should be a safe upgrade for callers already on
the 0.1.x line and is most relevant to applications using replayed graph
operators, durable memory/event-log components, snapshot stores, native credit
snapshots, embedded validation, or the examples facade.
