# manyfold 0.1.2

Release date: 2026-05-03

## Summary

manyfold 0.1.2 is a backward-compatible patch release for the local durable
component helpers. It hardens `FileStore` key addressing so special key parts
such as `.`, `..`, and empty strings remain inside the configured store root
instead of being interpreted as filesystem path control segments.

This release also includes the repository maintenance changes that landed after
0.1.1: a Manyfold-owned Rx import facade, new managed sensor run-loop helpers,
import grouping with ruff linting, and startup guidance for future automation
runs.

## What Changed

### Fixed `FileStore` special key-part escaping

`FileStore` and `Keyspace` now encode key parts through a small path-safe layer
before building filesystem paths. Regular key parts keep the same readable
percent-encoded representation, while ambiguous path segments are escaped:

- `.` is stored as `%2E`
- `..` is stored as `%2E%2E`
- an empty key part is stored as `%00`

Scans decode those path segments back to the original logical key parts, so
callers keep the same object-shaped `StoreEntry.key` values.

### Rejected NUL-containing key parts

Key normalization now rejects NUL bytes with `ValueError`. NUL is not a valid
filesystem path segment and would otherwise collide with the empty-key escape.

### Added regression coverage

The component test suite now covers traversal-looking key parts, empty key
parts, scan round-tripping, and NUL rejection.

### Exposed a Manyfold-owned Rx facade

The package now exposes `manyfold.rx` and compatibility submodules such as
`manyfold.rx.operators`, `manyfold.rx.subject`, `manyfold.rx.scheduler`, and
`manyfold.rx.testing.marbles`. These modules re-export the underlying RxPy
primitives through a project-owned import path so downstream code can depend on
Manyfold's streaming surface while the implementation continues to evolve.

### Added managed sensor run-loop helpers

The sensor IO surface now includes `ManagedRunLoop`, `ManagedRunLoopHandle`,
`ReactiveSensorSource`, and `ReactiveSensorHandle` exports. These helpers make
sensor polling and graph installation easier to own, stop, and test from the
Python API.

### Included maintenance since 0.1.1

- Grouped imports consistently across the Python package and tests.
- Added `uv run ruff check` to the verified command set.
- Added repository startup guidance in `AGENTS.md` for fresh detached worktrees.

## Why This Matters

`FileStore` is the byte-addressed backing layer used by higher-level durable
components such as `EventLog`, `SnapshotStore`, and `Memory`. Key parts are
logical data, not filesystem instructions. A key part of `..` previously passed
through `urllib.parse.quote(..., safe="")` unchanged, allowing writes below a
keyspace to escape to that keyspace's parent directory.

That behavior could corrupt unrelated component data or place files outside the
intended store subtree. The new encoding keeps logical keys and filesystem paths
separate.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- Existing ordinary key parts keep the same on-disk representation.
- Existing data using `.` or `..` as logical key parts may need manual migration
  from the old accidental filesystem locations into the new escaped paths.

## Versioning Decision

This release is tagged as `0.1.2` rather than `0.2.0` because it fixes incorrect
and unsafe behavior in an existing public API without expanding the API surface.

## Suggested GitHub Release Notes

### Highlights

- Fixed `FileStore` path traversal through special logical key parts.
- Added regression coverage for `.`, `..`, empty key parts, and NUL rejection.
- Added the `manyfold.rx` facade over RxPy streaming primitives.
- Included managed sensor run-loop and reactive source helpers.
- Included import/linting cleanup and automation startup guidance since 0.1.1.
- Bumped package version to `0.1.2`.

### Detailed Notes

manyfold `0.1.2` hardens the filesystem-backed `FileStore` used by durable
component helpers. Logical key parts are now encoded before they are appended to
filesystem paths, so values like `..` no longer escape the configured keyspace
or store root. Scans decode the escaped path parts back into the original
logical key tuples, preserving the public API shape.

The release also rejects NUL-containing key parts, adds focused component
regression tests, and carries forward the post-0.1.1 commits that added a
Manyfold-owned Rx facade, managed sensor run-loop helpers, ruff lint
verification, and startup guidance for fresh automation worktrees.
