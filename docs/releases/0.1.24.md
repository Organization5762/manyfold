# manyfold 0.1.24

Release date: 2026-05-06

## Summary

manyfold 0.1.24 is a backward-compatible patch release that packages the
post-0.1.23 validation, facade, durable component, route memory, sensor, and
native mailbox fixes. It focuses on clearer diagnostics, stable private Rx
exports, safer filesystem persistence, snapshot write ordering, consensus
quorum isolation, deterministic native topology queries, stable mailbox credit
lookups, clearer example helper exports, repeatable RFC checklist validation,
and documented local setup traps.

## What Changed

### Hardened durable components

- `FileStore` writes now replace values atomically, so a failed replacement keeps
  the previously durable value and cleans up temporary files.
- `EventLog` now ignores noncanonical numeric key names when reading records or
  allocating the next append index, avoiding accidental decode failures or
  sparse index jumps from stray keyspace entries.
- `Memory` records now use compact JSONL encoding while preserving the existing
  on-disk schema.
- `SnapshotStore` serializes writes, latest reads, persisted latest replay, and
  write-to-latest publication with a per-store lock. Concurrent publishers no
  longer allow an older write to persist or publish after a newer one.
- `Memory.resume()` now suppresses active `remember()` observers for the record
  currently being restored, avoiding duplicate JSONL appends during live resume.

### Improved consensus behavior

- The default consensus component now tracks votes by `(term, candidate)` so
  votes for another candidate do not leak into the local candidate's quorum.
- Confirmed quorum announcements emit one heartbeat per `(term, candidate)` and
  only confirmed heartbeat/quorum matches update leader state.

### Stabilized validation and facades

- Shared example helper exports are now explicit so catalog and reference
  examples continue to expose their intended helper surface.
- Example catalog duplicate-name errors now include enough location context to
  identify the colliding entry quickly.
- Duplicate shared example exports are removed from the generated catalog
  facade.
- Mailbox descriptor validation reports clearer route and mailbox configuration
  diagnostics.
- RFC appendix checklist parsing is stricter and more descriptive.
- RFC checklist generation now skips unchanged writes so check-mode validation
  does not dirty working trees with identical content.
- Stub graph query helpers now return sorted route and edge snapshots to match
  native query determinism in tests and pure-Python fallback paths.
- Private Rx subject facade exports are narrowed so internal compatibility
  boundaries remain explicit.
- Private Rx facade exports are stabilized at the package boundary so internal
  modules import from one compatibility surface.
- Threshold mapping comparisons are stable when sensor specs include threshold
  mappings with equivalent keys in different insertion orders.

### Improved local setup notes

- Agent startup notes now document the local uv cache workaround for environments
  where the default cache path is not writable.
- Agent startup notes now also call out rebuilding the editable native extension
  after Rust edits before running Python tests that import `_manyfold_rust`.

### Test stability

- Reactive coalescing coverage now uses a wider timing window to reduce
  scheduler jitter sensitivity while still asserting latest-value behavior.
- Background threaded observable teardown now disposes the source subscription
  and completes the yielded subject when leaving the context manager.

### Improved native runtime determinism

- Native topology query results are sorted for stable repeated inspection.
- Mailbox draining avoids unnecessary route clones.
- Mailbox credit lookup is stable when credit snapshots are queried by route.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing route, graph, component, consensus, sensor, and schema APIs remain
  compatible.
- New behavior fixes incorrect edge-case handling and improves diagnostics.

## Versioning Decision

This release is tagged as `0.1.24` because it groups several patch-level
correctness and validation fixes that landed after `0.1.23`, including durable
component and native mailbox behavior that should be available from PyPI. It
also publishes validation and setup-note improvements that reduce recurring
automation churn.
