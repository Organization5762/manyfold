# manyfold 0.1.28

Release date: 2026-05-09

## Summary

manyfold 0.1.28 is a backward-compatible patch release that makes durable
component key handling explicit. `FileStore`, `Keyspace`, and `SnapshotStore`
now define their accepted key-part domain instead of stringifying arbitrary
Python objects into persistent filesystem paths.

## What Changed

### Defined durable key-part validation

- `FileStore` and `Keyspace` now accept only string key parts and plain integer
  key parts.
- Booleans, floats, and arbitrary objects now raise `ValueError` instead of
  being coerced into durable path segments such as `True`, `1.5`, or an object
  `repr`.
- `SnapshotStore.key` now uses the same key-part validator as the byte store, so
  direct snapshot construction and keyspace operations agree on valid durable
  addresses.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing string and integer keys keep the same on-disk representation.
- Empty strings, `.`, `..`, and `__value__.bin` remain supported logical key
  parts and continue to be escaped by the existing filesystem-safe layer.
- Code that passed booleans, floats, or arbitrary objects as key parts now fails
  fast with `ValueError` rather than persisting ambiguous paths.

## Versioning Decision

This release is tagged as 0.1.28 because durable store address validation is a
user-visible behavior hardening change. It preserves supported string and
integer keys while turning previously implicit object coercion into a defined
runtime error.
