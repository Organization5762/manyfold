# manyfold 0.1.32

Release date: 2026-05-15

## Summary

manyfold 0.1.32 is a backward-compatible patch release that packages the
main-branch hardening work landed since 0.1.31. The release focuses on making
public construction paths, persisted JSON payloads, scheduler state, sensor IO,
query-plane records, component storage, and test support fail earlier with
stable validation errors.

The release also includes a handful of performance and determinism improvements:
ordered native name registries, cached route display and query sort keys,
fewer event-log record reads, reduced mailbox envelope cloning, and stable
metadata ordering across graph, sensor, component, catalog, and manifest
surfaces.

## What Changed

### Hardened graph and query-plane APIs

- Graph record dataclasses now validate taint repairs, route manifests,
  descriptors, audit/debug records, active flows, join plans, query requests,
  query responses, scheduled writes, snapshots, graph manifests, and diagram
  manifests more consistently.
- Taint repair construction now rejects fake enum-like domains, malformed added
  marks, non-tuple clear lists, blank cleared mark IDs, and non-string proofs.
- Query-plane responses reject blank item labels, and query principal handling
  now prevents mismatched requester/principal identities from crossing the
  query service boundary.
- Graph access capability wrappers and route-like helper inputs now reject
  malformed objects at construction time instead of failing later during graph
  operations.
- Manifest, diagram, route display, JSON rendering, and metadata ordering are
  deterministic for colliding or non-string mapping keys.
- `ReadThenWriteNextEpochStep.start()` now revalidates its connector result so
  an invalid subscription-like object fails immediately.

### Hardened sensors and runtime loops

- Sensor source, peripheral adapter, reactive source, debug tap, and publisher
  sequence counters no longer advance when graph publication fails.
- Mutable `ManualClock`, `SequenceCounter`, and sensor debug history state is
  revalidated at use time so post-construction mutation cannot bypass guards.
- Sensor JSON schemas reject missing required fields, invalid base64 payloads,
  non-finite numbers, non-standard JSON constants, malformed health/sample/event
  fields, invalid raw bytes, invalid nested byte markers, and malformed identity
  or location objects.
- JSON event decoding, sensor mappings, metadata restoration, tag metadata, and
  event metadata now preserve stable ordering.
- Frame assembly, checksum, filter callbacks, batching helpers, backoff,
  protobuf payloads, debug payloads, and sensor helper metadata gained earlier
  validation with focused diagnostics.

### Hardened components and storage

- Component schemas, component record containers, consensus inputs, route
  bundles, legacy consensus payloads, and non-standard component JSON constants
  now fail with explicit validation errors.
- Memory, file-store, event-log, durable spool, snapshot-store, and keyspace
  inputs now reject malformed roots, keys, record fields, schema versions, and
  store objects before side effects occur.
- File store roots and path-derived keys are validated more strictly, and
  process-local `Schema.any()` tokens now remain scoped by schema version while
  accepting bytes-like token payloads.
- Event-log and mailbox paths avoid unnecessary record reads, envelope clones,
  and unstable diagnostic ordering.

### Stabilized examples, catalogs, and tooling

- Example catalog module names, lookup keys, numeric metadata, implemented
  reference metadata, drift diagnostics, and malformed lookup coverage were
  tightened.
- Lego catalog lookup input handling and dependency-cycle checks were cleaned
  up and optimized.
- Project metadata checks now use TOML parsing, keep dependency and export
  ordering stable, restore module state on failed loads, and detect runtime
  `assert` statements in `python/manyfold`.
- Import-path rollback and stale RFC checklist diagnostics are documented and
  covered by tests.

### Runtime and packaging improvements

- Native route, mailbox, and schema name registries now have ordered lookup
  behavior.
- Route display formatting, native query sort keys, and primitive enum
  validation members are cached on hot paths.
- Reactive-thread helpers trim names, reset shutdown test state, validate helper
  names and latency stats, stabilize facade submodule bindings, and keep Rx
  operator facade exports tighter.
- Catalog, embedded, graph, sensor, primitive, stats, and reference helper
  public surfaces remain narrow, sorted, and explicit.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Valid callers should keep the same behavior.
- Malformed construction inputs, non-standard JSON constants, invalid mutable
  state, and fake enum-like objects now fail earlier with explicit exceptions.
- Some invalid payloads that previously decoded as `None`, `nan`, `inf`, or
  fake object-shaped records now raise `ValueError` or `PermissionError`.

## Versioning Decision

This release is tagged as 0.1.32 because four days of compatible hardening,
determinism, and validation work have accumulated since 0.1.31. The changes are
patch-level for valid callers, but the grouped batch is large enough that
packaging it is more useful than continuing to defer it.
