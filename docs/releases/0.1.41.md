# manyfold 0.1.41

manyfold 0.1.41 adds Rust-owned observability records and default
ManyFold-backed destinations for PubSub metrics and logs.

## Features

- Adds `ObservabilityRuntime` to the Rust/PyO3 layer for creating
  OTEL-compatible histogram metric records and structured log envelopes.
- Adds default Python `MetricsDestination` and `LogDestination` helpers that
  publish observability records through PubSub sidecar topics.
- Instruments PubSub publish calls with Rust-created payload-size,
  subscriber-delivery, and publish log records.
- Exposes recent PubSub observability records through `PubSub.observability_*`
  APIs for local inspection and tests.

## Runtime Notes

- The default log destination writes local rotating logs with a 100 MB file
  limit and 5 retained backups, then republishes collected log lines through
  the ManyFold observability namespace.
- The observability destinations are the first users of the ManyFold
  infrastructure path for metrics and logs; broader tracing and collector
  integration remain follow-up work.
- The old Reactivex dependency has been removed from the package metadata; the
  active stream surface is now the PubSub and DataStreamProcessor-oriented API.
