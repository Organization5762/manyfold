# manyfold 0.1.10

Release date: 2026-05-03

## Summary

manyfold 0.1.10 is a backward-compatible patch release for route memory
persistence. It prevents a fresh `Memory(path)` instance from duplicating an
already-persisted latest event when it subscribes to a graph with replay enabled.

## What Changed

### Fixed persisted memory duplicate suppression

`Memory.remember()` keeps a duplicate filter so replayed latest values are not
written to the JSONL memory file more than once. Before this release, that
filter only covered events observed by the current `Memory` instance. Creating a
new `Memory(path)` for an existing file could reappend the same latest event if
the graph replayed it during subscription.

The duplicate filter now seeds itself from the existing JSONL records at
construction time, so duplicate suppression survives process and object
restarts.

### Added regression coverage

The component tests now cover a fresh `Memory(path)` subscribing to a graph whose
latest event is already present on disk and assert that only one durable record
remains.

## Why This Matters

Route memory is intended to be a durable append log for replay and resume. Silent
duplicate records make resumed histories noisy and can cause downstream code to
reprocess the same event after a service restart or after rebuilding a memory
adapter around an existing file.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- Memory JSONL file format is unchanged.
- The only behavior change is stricter duplicate suppression for records that
  already exist on disk.

## Versioning Decision

This release is tagged as `0.1.10` rather than `0.2.0` because it fixes a
persistence bug in an existing public component without changing the public API.

## Suggested GitHub Release Notes

### Highlights

- Fixed `Memory(path)` to seed duplicate suppression from existing JSONL records.
- Prevented replay-enabled subscriptions from reappending already-persisted
  latest events after rebuilding a memory adapter.
- Added regression coverage for restart-safe memory duplicate filtering.
- Bumped package version to `0.1.10`.

### Detailed Notes

manyfold `0.1.10` fixes route memory persistence after object or process
restarts. Existing memory records now populate the duplicate filter when a
`Memory` instance is created, preserving the JSONL format while avoiding
duplicate durable events.
