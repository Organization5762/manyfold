# manyfold 0.1.5

Release date: 2026-05-03

## Summary

manyfold 0.1.5 is a backward-compatible patch release for the durable
`Memory` component. It fixes duplicate suppression after a graph or process
restart so fresh events are not dropped merely because a new graph reused the
same per-route sequence number as a previously persisted event.

## What Changed

### Fixed `Memory` restart recording

`Memory.remember()` no longer loads prior records as a durable duplicate filter.
It now keeps duplicate detection scoped to the live recorder instance and keys
those live observations by route, source sequence, serialized payload, and
control epoch. Previous releases loaded route/source-sequence pairs from disk as
if they were globally stable. That was too broad for durable memory because
`Graph` source sequence numbers start again from 1 in a new graph instance.

Before this release:

- a first graph could publish and persist `route@1 = b"first"`
- a restarted graph could publish a fresh `route@1`
- `Memory` loaded from disk considered the fresh event already seen and skipped
  it, even if it was the first event in a new process

After this release:

- exact duplicate observations seen by the same live recorder are still ignored
- fresh restarted-graph events are recorded even when route sequence numbers
  repeat

### Added regression coverage

The component test suite now covers recording a route across two graph
instances where both graphs emit source sequence 1 and the same payload. The
test asserts that both durable events are retained in order while existing
duplicate-subscription coverage continues to pass.

## Why This Matters

`Memory` is the durable route recorder used to resume typed graph values from
disk. Restarted processes and short-lived graph instances are normal usage
patterns for local device and automation code. Dropping the first fresh event
after restart could lose state updates, sensor readings, or command history
without raising an error.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- Existing JSONL memory files remain readable.
- Duplicate suppression is narrower: only exact route, sequence, payload, and
  control-epoch matches observed by the same live recorder are treated as
  already seen.

## Versioning Decision

This release is tagged as `0.1.5` rather than `0.2.0` because it fixes incorrect
behavior in an existing public API without expanding the API surface.

## Suggested GitHub Release Notes

### Highlights

- Fixed `Memory` dropping fresh restarted-graph events with reused source
  sequence numbers.
- Preserved duplicate suppression for exact repeated observations within one
  live recorder.
- Added regression coverage for sequence reuse across graph instances.
- Bumped package version to `0.1.5`.

### Detailed Notes

manyfold `0.1.5` fixes durable `Memory` duplicate detection. The recorder no
longer treats route source sequence numbers from existing JSONL records as
globally stable event identities. Live duplicate detection now uses the route
display, source sequence, encoded payload, and control epoch, which keeps
repeated subscriptions from writing identical records while allowing restarted
graphs to persist new events even when their source sequence counters begin
again at 1.
