# manyfold 0.1.14

Release date: 2026-05-03

## Summary

manyfold 0.1.14 is a backward-compatible patch release for the filesystem-backed
component store. It fixes an internal filename collision so user key segments can
literally be named `__value__.bin` without blocking parent values or nested
values under the same keyspace.

## What Changed

### Reserved the FileStore value sentinel

`FileStore` now percent-encodes key parts that would collide with the internal
`__value__.bin` payload filename. This allows a keyspace to store both its own
value and a child key whose segment is also `__value__.bin`.

`Keyspace.get(...)` and `Keyspace.delete(...)` now treat directories at the value
path as missing values instead of trying to read or unlink them as files.

`Keyspace.scan(...)` now orders results by decoded full key, keeping listing
order stable and logical even when reserved key parts need internal escaping.

### Added regression coverage

The component tests now cover storing and scanning a parent key alongside a child
key named `__value__.bin`.

## Why This Matters

The component store is a durable byte-addressed substrate used by higher-level
logs, snapshots, and examples. User-controlled key parts should not be able to
collide with implementation sentinels or make unrelated parent keys unreadable.

## Compatibility

This is a patch release.

- No public APIs were removed or renamed.
- No call signatures changed.
- Existing keys with ordinary segments keep the same on-disk layout.
- The reserved segment `__value__.bin` now uses a safer escaped layout for new
  writes.

## Versioning Decision

This release is tagged as `0.1.14` rather than `0.2.0` because it fixes a
backward-compatible persistence edge case without changing public APIs.

## Suggested GitHub Release Notes

### Highlights

- Escaped the internal FileStore value sentinel when it appears as a user key
  part.
- Made `get` and `delete` treat directory collisions as missing values.
- Sorted scans by decoded full key for stable logical ordering.
- Bumped package version to `0.1.14`.

### Detailed Notes

manyfold `0.1.14` hardens the filesystem component store against reserved
filename collisions. Parent values and nested values can now coexist even when a
user key segment is literally `__value__.bin`.
