# manyfold 0.1.30

Release date: 2026-05-09

## Summary

manyfold 0.1.30 is a backward-compatible patch release that defines event-time
operator behavior that was previously implicit. Watermark progress now has
idempotent release semantics, custom progress extractors fail fast when they
return ambiguous values, and interval joins can use explicit event-time
extractors instead of only per-route sequence numbers.

This release also consolidates the already-published 0.1.28 and 0.1.29
hardening changes with current main so the new version does not regress durable
key validation, native-boundary validation, or object-shaped graph manifests.

## What Changed

### Defined event-time progress

- `window_by_time()` and `window_aggregate_by_time()` now require custom
  `event_time` and `watermark_time` extractors to return non-negative integers.
- Boolean, string, float, and negative progress values now raise
  `ValueError("event-time progress must be a non-negative integer")` instead of
  being compared or subtracted later.
- Default progress remains unchanged: `control_epoch` is used when present and
  `seq_source` is used otherwise.

### Stabilized watermark releases

- Duplicate explicit watermarks are now idempotent when no source partition has
  accepted new data since the last release.
- Late source data that is still inside grace marks its partition dirty, so a
  repeated current watermark emits the updated window once.
- Source events older than the retained watermark window plus grace continue to
  be dropped.

### Added event-time interval joins

- `Graph.interval_join()` now accepts `left_time=` and `right_time=` extractors
  for typed values or raw bytes.
- Interval joins retire buffered events behind a monotonic progress frontier,
  making late-beyond-window behavior explicit.
- The existing sequence-based behavior remains the default when no extractors
  are provided.

### Consolidated release history

- Durable `FileStore` key-part validation from 0.1.28 is included.
- Native PyO3 reference and control-epoch validation from 0.1.29 is included.
- Object-shaped `Graph.manifest()` and JSON rendering from current main are
  included.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing valid event-time windows and interval joins keep their default
  behavior.
- Duplicate watermarks without new data no longer produce repeated identical
  windows.
- Code whose custom progress extractors returned booleans, strings, floats, or
  negative integers now fails with a stable `ValueError`.

## Versioning Decision

This release is tagged as 0.1.30 because 0.1.28 and 0.1.29 are already
published GitHub releases. The new behavior is user-visible runtime hardening,
but it remains compatible for valid callers.
