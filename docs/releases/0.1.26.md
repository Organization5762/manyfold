# manyfold 0.1.26

Release date: 2026-05-06

## Summary

manyfold 0.1.26 is a backward-compatible patch release for fixes that landed
after 0.1.25. It makes native mailbox edges behave like the documented queue
boundary, ignores stale coalesce timers, and clarifies native rebuild guidance
after rebases.

## What Changed

### Fixed native mailbox routing

- Native mailbox edges now enqueue writes into the mailbox queue instead of
  writing only to the ingress route metadata.
- Mailbox draining now publishes queued messages from the mailbox egress route,
  so downstream consumers receive messages through the advertised mailbox
  boundary.
- Mailbox credit snapshots now report the largest queue depth actually reached
  instead of using configured capacity as the high-water mark.

### Stabilized coalescing and setup guidance

- Coalescing pipeline nodes now ignore stale timer callbacks after a newer
  pending value schedules a later flush.
- Agent startup notes now call out rebuilding the editable native extension
  after rebases that include Rust changes.

## Compatibility

This is a patch release.

- No public APIs were removed.
- Existing route, graph, mailbox, producer, pipeline, and flow snapshot APIs remain
  compatible.
- New behavior fixes incorrect native mailbox delivery, credit metrics, and
  stale coalescing timer behavior.

## Versioning Decision

This release is tagged as 0.1.26 because 0.1.25 was already tagged before the
native mailbox queue-routing fix landed. The fix affects runtime delivery
semantics and should be available from PyPI as a focused follow-up patch.
